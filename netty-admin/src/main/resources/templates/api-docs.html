API 参考
数据接入控制台对外暴露的 REST 接口
BaseURL: http://host:2310
协议管理接口
POST /protocols/reload
重新扫描 protocols/ 目录下的所有 *.jar，热加载新协议。旧连接不受影响，新连接走新版本 Provider。
GET /protocols
列出所有已注册协议 + 端口绑定 + 每端口活跃连接数 + 顶部统计概览。
DELETE /protocols/{name}
卸载协议。框架会自动关闭该协议绑定的所有端口监听、强制断开所有活跃连接、调 Provider.destroy()、释放 URLClassLoader 与 jar 文件句柄。
POST /protocols/{name}/bind/{port}
绑定协议到端口，触发 Netty 端口监听。若端口已在监听，会先关闭再重新绑定确保 Pipeline 走新协议。
DELETE /protocols/bind/{port}
解绑端口。关闭端口监听 + 踢掉所有客户端连接 + 移除端口→协议映射。
调用示例（curl） 
# 热加载协议 jar
curl -X POST http://localhost:2310/protocols/reload

# 查看当前状态
curl http://localhost:2310/protocols

# 绑定 modbus-tcp 协议到 502 端口
curl -X POST http://localhost:2310/protocols/modbus-tcp/bind/502

# 解绑端口（关闭监听并踢连接）
curl -X DELETE http://localhost:2310/protocols/bind/502

# 卸载协议（自动清理所有相关端口与连接）
curl -X DELETE http://localhost:2310/protocols/modbus-tcp
响应格式约定

所有接口统一返回 JSON，包含以下字段：

  • success：布尔，操作是否成功
  • reason：失败时的人类可读原因（仅 success=false 时出现）
  • message：成功时的描述信息（可选）
  • closedPorts：卸载协议时被自动关闭的端口列表
  • loaded：reload 时各 Provider 的加载结果列表
