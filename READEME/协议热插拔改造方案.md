# Netty 协议热插拔架构改造

## Context

**问题**：当前 `NettyChannelInitializer.java:74` 硬编码 `channelPipeline.addLast("hj212Decoder", new HJ212Decoder())`，新增协议必须改核心代码、重新打包、重启服务。

**目标**：一次改造后，未来新增协议的流程为「写协议实现 jar → 放到 `protocols/` 目录 → 调 HTTP 接口」即可生效，**核心代码零修改、服务零重启**。

**技术路径**：动态 `URLClassLoader` + Java SPI（`ServiceLoader`）+ HTTP 管理接口。

**用户选型决策**：
- 路由维度：按服务端口（一个端口绑一种协议）
- 热加载触发：HTTP 接口手动 reload
- 卸载：支持（先解绑端口再卸载）
- Handler 实例：每连接新实例（Provider 工厂方法）
- 兼容性：保留旧 `HJ212Decoder` 类不动，新建 `HJ212ProtocolProvider` 包装

## 架构图

```
┌──────────────────────────────────────────────────────────┐
│ HTTP 控制面（端口 2310，复用 SpringBoot 嵌入式 Tomcat） │
│ POST   /protocols/reload               重新扫描 jar 目录 │
│ GET    /protocols                       列出已加载协议   │
│ DELETE /protocols/{name}                卸载协议          │
│ POST   /protocols/{name}/bind/{port}    绑定到端口        │
│ DELETE /protocols/bind/{port}           解绑端口          │
└────────────────┬─────────────────────────────────────────┘
                 │
   ┌─────────────▼──────────────┐   ┌─────────────────────────┐
   │ ProtocolRegistry（单 Bean）│◄──┤ ProtocolJarLoader        │
   │ protocols  Map<name, LP>   │   │ - 扫描 ./protocols/*.jar│
   │ portBindings Map<port,name>│   │ - URLClassLoader per jar │
   │ register/unregister/get    │   │ - ServiceLoader 发现    │
   └─────────────┬──────────────┘   └─────────────────────────┘
                  │
   ┌──────────────▼──────────────────────────────────────────┐
   │ ProtocolDecoderProvider（SPI 接口，内置+外部同接口）    │
   │ name() / version() / description() / createHandlers()  │
   │ idleConfig() / destroy()                                │
   └──────────────┬──────────────────────────────────────────┘
                  │
   ┌──────────────▼─────────────────────────────────────────┐
   │ NettyChannelInitializer（改造：不再硬编码 HJ212）       │
   │ 1. serverPort = socketChannel.localAddress().getPort() │
   │ 2. protoName = registry.getProtocolNameByPort(port)    │
   │ 3. provider  = registry.getProvider(protoName)         │
   │ 4. pipeline.addLast([IdleStateHandler] + handlers[])   │
   │ 兜底：未匹配 → close + 错误日志                         │
   └────────────────────────────────────────────────────────┘
```

### Step 3：新增 SPI 接口契约

**【新增】** `src\main\java\com\cas\access\netty\protocol\ProtocolDecoderProvider.java`

```java
package com.cas.access.netty.protocol;

import io.netty.channel.ChannelHandler;

public interface ProtocolDecoderProvider {
    String name();
    default String version() { return "1.0.0"; }
    default String description() { return ""; }

    /**
     * 为一条新连接创建 Handler 链。
     * 框架在 IdleStateHandler 之后逐个 addLast。
     * 实现必须返回新实例（除非 Handler 标注 @ChannelHandler.Sharable）。
     */
    ChannelHandler[] createHandlers();

    default IdleConfig idleConfig() { return null; }
    default void destroy() { }

    final class IdleConfig {
        public final int readIdleSeconds, writeIdleSeconds, allIdleSeconds;
        public IdleConfig(int r, int w, int a) {
            this.readIdleSeconds = r; this.writeIdleSeconds = w; this.allIdleSeconds = a;
        }
    }
}
```

### Step 5：新增 HJ212 包装 Provider

**【新增】** `src\main\java\com\cas\access\netty\handler\HJ212ProtocolProvider.java`

```java
package com.cas.access.netty.handler;

import com.cas.access.netty.protocol.ProtocolDecoderProvider;
import io.netty.channel.ChannelHandler;
import io.netty.handler.codec.string.StringEncoder;
import org.springframework.stereotype.Component;

import java.nio.charset.StandardCharsets;

@Component
public class HJ212ProtocolProvider implements ProtocolDecoderProvider {

    private final ReadEventHandler readEventHandler;   // @Sharable @Component，注入

    public HJ212ProtocolProvider(ReadEventHandler readEventHandler) {
        this.readEventHandler = readEventHandler;
    }

    @Override public String name()        { return "hj212"; }
    @Override public String version()     { return "1.0.0"; }
    @Override public String description() { return "HJ212 环保传输协议（内置）"; }

    @Override
    public ChannelHandler[] createHandlers() {
        // HJ212Decoder 必须 new（ByteToMessageDecoder 有状态）；ReadEventHandler 是 @Sharable 可共享
        return new ChannelHandler[]{
            new HJ212Decoder(),
            readEventHandler,
            new StringEncoder(StandardCharsets.UTF_8)
        };
    }

    @Override public IdleConfig idleConfig() { return new IdleConfig(600, 600, 600); }
}
```

**HJ212Decoder、ReadEventHandler 类完全不动。**

### Step 6：新增 ProtocolRegistry（注册中心）

**【新增】** `src\main\java\com\cas\access\netty\protocol\ProtocolRegistry.java`

```java
package com.cas.access.netty.protocol;

import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;

import java.util.Collection;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

@Slf4j
@Component
public class ProtocolRegistry {

    private final Map<String, LoadedProtocol> protocols = new ConcurrentHashMap<>();
    private final Map<Integer, String> portBindings = new ConcurrentHashMap<>();

    public LoadedProtocol getProvider(String name) {
        LoadedProtocol p = protocols.get(name);
        return (p != null && p.isActive()) ? p : null;
    }

    public Collection<LoadedProtocol> listAll() { return protocols.values(); }

    /** 同名协议：先卸载旧的（关 ClassLoader）再注册新的 */
    public synchronized void register(LoadedProtocol lp) {
        LoadedProtocol old = protocols.get(lp.getName());
        if (old != null) {
            log.warn("协议[{}]被覆盖: {} → {}", lp.getName(), old.getVersion(), lp.getVersion());
            unregisterInternal(lp.getName());
        }
        protocols.put(lp.getName(), lp);
    }

    /** 卸载：先校验端口绑定，仍被绑定则拒绝 */
    public synchronized boolean unregister(String name) {
        if (portBindings.containsValue(name)) {
            log.error("协议[{}]仍被端口绑定，拒绝卸载", name);
            return false;
        }
        return unregisterInternal(name);
    }

    private boolean unregisterInternal(String name) {
        LoadedProtocol lp = protocols.remove(name);
        if (lp == null) return false;
        try { lp.getProvider().destroy(); } catch (Throwable t) { log.warn("destroy 异常: {}", t.getMessage()); }
        if (lp.getClassLoader() != null) {
            try { lp.getClassLoader().close(); }   // Windows 文件锁释放
            catch (Exception e) { log.warn("ClassLoader close 失败: {}", e.getMessage()); }
        }
        log.info("协议[{}]已卸载", name);
        return true;
    }

    public void bindPortToProtocol(int port, String name) {
        if (!protocols.containsKey(name)) throw new IllegalArgumentException("协议未注册: " + name);
        portBindings.put(port, name);
    }
    public String unbindPort(int port) { return portBindings.remove(port); }
    public String getProtocolNameByPort(int port) { return portBindings.get(port); }
    public Map<Integer, String> getAllBindings() { return new java.util.TreeMap<>(portBindings); }
}
```

### Step 7：新增 ProtocolJarLoader（动态加载器）

**【新增】** `src\main\java\com\cas\access\netty\protocol\ProtocolJarLoader.java`

核心逻辑：
- 扫 `${netty.server.protocol.jar-dir}/*.jar`
- 每个 jar 一个独立 `URLClassLoader`（parent = 当前线程 ContextClassLoader，确保能引用到主项目的 netty 与 SPI 接口）
- `ServiceLoader.load(ProtocolDecoderProvider.class, cl)` 发现实现
- 包装为 `LoadedProtocol` 注册到 Registry
- 若 jar 中无任何 Provider 实现，立即 `cl.close()` 避免泄漏

返回 `List<LoadResult>` 给 HTTP 接口反馈加载情况（含每个 jar 的 provider 列表与状态）。

### Step 9：新增 HTTP 控制器

**【新增】** `src\main\java\com\cas\access\netty\protocol\ProtocolController.java`

```java
@Slf4j
@RestController
@RequestMapping("/protocols")
public class ProtocolController {

    @Resource private ProtocolJarLoader jarLoader;
    @Resource private ProtocolRegistry registry;

    @PostMapping("/reload")
    public Map<String, Object> reload() { /* 调 jarLoader.scanAndLoad() */ }

    @GetMapping
    public Map<String, Object> list() { /* 返回 protocols + bindings */ }

    @DeleteMapping("/{name}")
    public Map<String, Object> unload(@PathVariable String name) {
        boolean ok = registry.unregister(name);
        // 失败时返回 {"success":false,"reason":"仍被端口绑定，请先 DELETE /protocols/bind/{port}"}
    }

    @PostMapping("/{name}/bind/{port}")
    public Map<String, Object> bindPort(@PathVariable String name, @PathVariable int port) {
        // 顺序很重要：先 registry.bindPortToProtocol，再 NettyServerUtil.bindPort
        // 避免 bind 后瞬间连接进来时映射还没建立
        if (registry.getProvider(name) == null) return fail("协议未注册");
        if (GlobalCache.SP_ServerChannel_Map.containsKey(port)) {
            NettyServerUtil.closeListen(port);   // 复用现有工具
        }
        registry.bindPortToProtocol(port, name);
        NettyServerUtil.bindPort(port);          // 复用 NettyServerUtil.java:24
        return ok();
    }

    @DeleteMapping("/bind/{port}")
    public Map<String, Object> unbindPort(@PathVariable int port) {
        NettyServerUtil.closeListen(port);       // 关闭监听 + 踢所有客户端连接
        registry.unbindPort(port);
        return ok();
    }
}
```

### Step 10：改造 NettyChannelInitializer

**【修改】** `src\main\java\com\cas\access\netty\server\NettyChannelInitializer.java`

关键改动：
- **移除** `@Resource ReadEventHandler`（改由 HJ212ProtocolProvider 注入）
- **新增** `@Resource ProtocolRegistry protocolRegistry`
- initChannel 逻辑改为「按端口查协议 → 取 Provider → createHandlers() → addLast」

```java
@Resource
private ProtocolRegistry protocolRegistry;

@Override
protected void initChannel(SocketChannel socketChannel) {
    ChannelPipeline pipeline = socketChannel.pipeline();
    int serverPort = socketChannel.localAddress().getPort();
    String clientIp = socketChannel.remoteAddress().getAddress().getHostAddress();
    int clientPort = socketChannel.remoteAddress().getPort();

    try {
        String protoName = protocolRegistry.getProtocolNameByPort(serverPort);
        LoadedProtocol loaded = protoName == null ? null : protocolRegistry.getProvider(protoName);

        if (loaded == null || !loaded.isActive()) {
            log.error("[拒绝连接] 端口 {} 未绑定可用协议，client={}:{}", serverPort, clientIp, clientPort);
            socketChannel.close();
            return;
        }

        ProtocolDecoderProvider provider = loaded.getProvider();
        ProtocolDecoderProvider.IdleConfig idle = provider.idleConfig();
        int r = idle == null ? 600 : idle.readIdleSeconds;
        int w = idle == null ? 600 : idle.writeIdleSeconds;
        int a = idle == null ? 600 : idle.allIdleSeconds;
        pipeline.addLast("idleStateHandler", new IdleStateHandler(r, w, a));

        ChannelHandler[] handlers = provider.createHandlers();
        if (handlers != null) {
            for (int i = 0; i < handlers.length; i++) {
                pipeline.addLast(provider.name() + "_" + i, handlers[i]);
            }
        }
        log.info("[Pipeline装配] client={}:{} port={} protocol={}", clientIp, clientPort, serverPort, protoName);
    } catch (Exception e) {
        log.error("Init Pipeline 失败 client={}:{} port={}", clientIp, clientPort, serverPort, e);
        socketChannel.close();
    }
}
```

## 风险与边界

| 风险 | 缓解 |
|---|---|
| **Windows 文件锁** | `unregister` 内调用 `URLClassLoader.close()` 释放 jar 句柄 |
| **外部 jar 带 @Component 污染主上下文** | 外部协议包名禁用 `com.cas.access`；外部 jar 禁用 Spring 注解；主类 `@SpringBootApplication` 已限定包扫描 |
| **ByteToMessageDecoder 非 @Sharable** | SPI 契约要求 `createHandlers()` 每次返回新实例；代码 review 检查 |
| **外部 jar 重复打包 netty-all** | 外部 jar 的 pom.xml 中 netty-all 用 `<scope>provided</scope>` |
| **initChannel 与 unregister 并发** | `LoadedProtocol.active` volatile，捕获 Throwable 失败即 close |
| **bindPort 顺序竞态** | 先 `bindPortToProtocol` 再 `bindPort`，避免端口绑定瞬间映射缺失 |
| **运维误删在用协议** | `unregister` 检查 `portBindings.containsValue`，仍被绑定则拒绝 |

### 外部 jar 开发者必读约定
1. netty-all、ProtocolDecoderProvider 接口都用 `<scope>provided</scope>`
2. 实现类**禁用** Spring 注解（@Component 等）
3. 必须包含 `META-INF/services/com.cas.access.netty.protocol.ProtocolDecoderProvider`，每行一个实现类全名
4. `createHandlers()` 返回新实例（除非显式 `@ChannelHandler.Sharable`）
5. `name()` 全局唯一，建议反向域名风格如 `com.acme.modbus`
